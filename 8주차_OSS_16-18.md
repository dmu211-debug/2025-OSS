# 오픈소스소프트웨어(OSS) 원격 수업 16~18 정리

## 16. 원격 저장소 복제(Clone)

### 원격 저장소 생성과 복제
- `git clone [복사된-주소]` : 원격 저장소와 동일한 이름으로 복제
- `git clone [복사된-주소] [새로운-폴더명]` : 하부 폴더 이름으로 복제
- `git clone [복사된-주소] .` : 현재 폴더에 바로 복제

### 원격 저장소 별칭 이름 점검
- `git remote` : 원격 저장소 목록 확인 (기본 이름 origin)
- `git remote -v` : 원격 저장소 주소 등 정보 표시

### 원격 저장소 별칭 관리
- `git remote add origin URL` : 원격 저장소 별칭 추가
- `git remote show origin` : 자세한 정보 확인
- `git remote rename origin org` : 이름 수정
- `git remote rm org` : 삭제

### 원격 저장소 정보 확인
- `git remote show origin` : 원격 저장소의 모든 브랜치 확인 가능

### VSCode로 원격 저장소 관리
- **Clone Git Repository** 클릭

## 17. 지역과 원격 저장소 연동 (push/pull)

### 원격 저장소 복제 및 연동
- `git clone http://github.com/atom/atom.git`

### 원격 저장소 수정 사항 가져오기
- `git pull origin main` : 원격 저장소 main 브랜치 변경 사항을 로컬로 가져오기
- `git pull` : 기본 원격 저장소 및 브랜치에서 가져오기

### 지역 저장소에서 push
- 쓰기 권한 필요
- `git push <저장소별칭명> <브랜치명>` : 원격으로 변경 사항 전송
- 예: `git push origin topic` : 최초 한 번 브랜치 지정 후 이후 간단히 `git push` 가능

### git pull 명령
- 내부적으로 `fetch` + `merge` 순차 수행
- **fetch**: 원격 저장소 정보를 로컬(remote/origin) 저장소로 가져오기
- **merge**: 로컬 저장소와 변경 사항 병합

### fetch
- 소스만 가져오고 병합은 미수행
- `git fetch <remote>` : 예) `git fetch origin`
- fetch 후 `origin/main`, `origin/HEAD`가 최신 위치
- 로컬 HEAD/main과 병합하려면 `git merge origin/main`
