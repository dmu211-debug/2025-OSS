# 오픈소스소프트웨어(OSS) 원격 수업 13~15 정리

## 13. 버전과 태그 활용

### 버전(version)
- 프로그램을 수정하거나 개선할 때마다 코드를 구분하기 위해 부여된 식별자
- 보통 숫자를 사용, '년,월'을 사용하기도 함
- Git SW 버전 예: 2.39.0

### SemVer(Semantic Versioning) 방식
- 형식: `major.minor.patch`
    - **Major**: 첫 자리가 0이면 초기 개발, 정식 버전은 1부터 시작
    - **Minor**: 메이저 버전 내 기능 추가나 변경 시 증가
    - **Patch**: 버그 수정 등 미미한 변화 시 증가

### 태그(tag) 기능
- 특정 커밋에 버전 번호 또는 이름 부여
    - 예: `v1.4.0`
- **주석 태그(Annotated tag)**: 태그 이름 + 작성자 이메일, 시각, 메시지 포함
- **일반 태그(Lightweight tag)**: 태그 이름만 포함

### 태그 생성
- **주석 태그**
  - `git tag -a v1.0.0 -m 'first version'` : 메시지 포함
  - `git tag -a v1.1.0 commitID` : 특정 커밋에 태그
- **일반 태그**
  - `git tag v1.0.1` : 이름만 지정, 중복 불가, -a/-m 사용 불가

### 태그 확인 및 삭제
- `git tag` : 태그 목록 확인
- `git show v1.0.0` : Annotated 태그 정보 확인
- `git tag -d v1.0.0` : 태그 삭제

## 14. 브랜치 개요와 관리

### 깃 브랜치 개념
- 버전 관리 중 파일 집합을 독립적으로 복사하여 개발 가능
- 여러 개발자가 동시에 다양한 작업 가능
- 브랜치 병합(merge): 독립 브랜치 작업 후 원래 브랜치와 합침

### 브랜치 장점
- 다른 브랜치 영향 없이 기능 개발, 버그 수정, 아이디어 실험 가능


### 기본 브랜치
- 저장소 생성 시 처음 생성
- 변경: `git config --global init.defaultBranch main`
- 기존 저장소 브랜치 이름 변경: `git branch -M newBname`

### 브랜치와 HEAD
- 브랜치: 커밋을 가리키는 포인터
- HEAD: 작업 중인 브랜치의 최신 커밋 포인터
- checkout/switch 명령으로 HEAD 이동 가능

### 브랜치 생성
- 생성만: `git branch bname`
- 생성 + HEAD 이동: `git switch -c bname` 또는 `git checkout -b bname`

### 브랜치 확인
- `git branch` : 브랜치 목록
- `git branch -v` : 마지막 커밋 ID 및 메시지 포함

### 브랜치 이동
- 특정 브랜치 이동: `git switch [bname]` 또는 `git checkout [bname]`
- 이전 브랜치로 이동: `git switch -` 또는 `git checkout -`

### 분리된 HEAD
- 이전 커밋 이동: `git checkout HEAD~`  
- 상대적 이동: `HEAD~`, `HEAD^^`, commitID 사용

### 브랜치 삭제
- 병합된 브랜치 삭제: `git branch -d branchName`
- 병합 안된 브랜치 강제 삭제: `git branch -D branchName`

### 브랜치 병합 상태 확인
- `git branch --merged` : 현재 브랜치 기준 병합된 브랜치
- `git branch --no-merged` : 현재 브랜치 기준 아직 병합되지 않은 브랜치
- 특정 브랜치 기준:
  - `git branch --merged branchName`
  - `git branch --no-merged branchName`
