## 19. 오픈소스 소프트웨어(OSS)

### OSS 정의
- OSS(Open Source Software) : 누구나 제한 없이 코드 열람, 수정, 재배포 가능한 소프트웨어
- OSI(Open Source Initiative)
  - opensource.org에서 공개 소스 정의(OSD) 관리
  - OSI 인증 마크 = 공개 소스임을 증명

### 자유 소프트웨어(Free Software)
- Richard Stallman, GNU 프로젝트, 자유 소프트웨어 재단(FSF)
- Copyleft : 저작권은 인정하지만 누구나 복제 및 수정 가능
- 예: Free BSD, GNU, Linux 기반 OS

### 오픈소스 방식의 의미
- 협업과 공유 중심
  - GitHub, GitLab, Bitbucket 등을 통한 소스코드 관리
  - 사용자와 개발자가 코드 공유, 수정, 재배포 가능
  - 대규모 협업과 지속 가능한 소프트웨어 개발 지원

### OSS 장단점
- 장점
  - 연구, 학습, 커스터마이징, 커뮤니티 기여 가능
  - 혁신 지원
- 단점
  - 공개 의무
  - 품질보증, 유지보수, 보안 어려움

### 대표 OSS 프로젝트
- OS : Linux, Android
- 웹 서버 : Apache
- DBMS : MySQL
- 프로그래밍 언어 : PHP, Python, Scikit-learn, TensorFlow, PyTorch
- 웹 브라우저 : Mozilla Firefox
- 오피스 : OpenOffice, LibreOffice
- 버전 관리 : Git

### 오픈소스 라이선스
- 저작권 범위 내 사용, 위반 시 법적 책임

#### 1. GPL(GNU General Public License)
- 자유소프트웨어재단(FSF) 라이선스
- 수정 후 재배포 시 GPL 공개 필수
- 적용: Linux, Git, MariaDB, WordPress, Firefox(v2.0)

#### 2. AGPL(GNU Affero GPL)
- GPL 기반, 네트워크 상호작용 소스코드도 공개
- 적용: MongoDB 등

#### 3. LGPL(Lesser GPL)
- GPL보다 완화, 소스 재배포 부담 완화
- 적용: Firefox(v2.1)

#### 4. Apache License
- 소스 공개 의무 없음
- 수정 배포 시 Apache License 포함 및 명시 필요
- 적용: Android, Hadoop

#### 5. MIT License
- 최소 제한 조건, 자유로운 수정 및 사용 가능
- 적용: Bootstrap, Angular.js, Backbone.js, jQuery

## 20. 임시저장(Stash)

### Stash 구조
- Git 임시저장소, 스택 구조
- 최신 내용이 가장 위에 저장

### 필요성
- 브랜치 이동, 이전 커밋 이동 시 작업 내용 저장 필요
- 작업 디렉토리 + 스테이징 영역 내용 저장
# 오픈소스소프트웨어(OSS) 원격 수업 19~21 정리

### Stash 명령
- 저장
  - `git stash`
  - `git stash -m '메시지'`
  - `git stash save`
  - `git stash save '메시지'`
- 옵션
  - `--keep-index (-k)` : 스테이징 제외
  - `--include-untracked (-u)` : Untracked 포함
  - `--patch (-p)` : 대화형 저장

### 복원
- `git stash apply` : 최신 stash 내용 적용
- `git stash apply stash@{n}` : 특정 stash 적용
- `git stash apply --index` : 스테이징 영역 포함 적용

### 목록 확인
- `git stash list`

### 차이 확인
- `git stash show`
- `git stash show -p`
- `git stash show stash@{n}`
- `git stash show stash@{n} -p`

### 삭제
- `git stash pop` : 적용 후 삭제
- `git stash pop stash@{n}`
- `git stash drop` : 최근 삭제
- `git stash drop stash@{n}` : 지정 삭제
- `git stash clear` : 전체 삭제

### Untracked 파일 삭제
- `git clean` : 안전 삭제
- `git clean -i` : 대화형
- `git clean -f` : 강제 삭제
