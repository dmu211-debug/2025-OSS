# 오픈소스소프트웨어(OSS) 원격 수업 10~12 정리

## 10. 파일 비교(diff)

### 스테이징 영역 기준으로 작업 디렉토리 비교
- `git diff` : 스테이징 영역과 작업 디렉토리의 차이 확인

### 깃 저장소(HEAD) 기준으로 스테이징 영역 비교
- `git diff --staged [HEAD]` : 스테이징 영역과 마지막 커밋(HEAD) 비교

### 깃 저장소(HEAD) 기준으로 작업 디렉토리 비교
- `git diff HEAD` : 작업 디렉토리와 마지막 커밋 비교  
- `git diff HEAD~2` : HEAD~2 기준으로 작업 디렉토리 비교

### 커밋 간 파일 비교
- `git diff [commit1] [commit2]` : 두 커밋 간 차이 비교  
  - 예: `git diff HEAD^ HEAD`

## 11. 파일 삭제(rm)과 복원(restore)

### 리눅스 명령으로 파일 삭제
- `rm [file]` : 작업 디렉토리에서 파일 삭제

### Git 명령으로 파일 삭제
- `git rm [file]`  
  - 작업 디렉토리 + 스테이징 영역에서 파일 삭제  
  - 다음 커밋에서 파일 제거됨, Tracked → Untracked 상태
- `git rm --cached [file]`  
  - 스테이징 영역에서만 삭제, 작업 디렉토리에는 남음  
  - `git ls-files` 결과에 나타나지 않음

### 파일 복원(restore)
- `git restore [file]`  
  - 작업 디렉토리의 파일을 스테이징 영역 상태로 복구  
  - 주의: 작업 디렉토리 내용이 덮어쓰기 됨
- `git restore --staged [file]`  
  - 스테이징 영역의 파일을 마지막 커밋 상태로 복구  
  - 주의: 스테이징 영역 내용이 사라짐
- `git restore --source=HEAD --staged --worktree [file]`  
  - 스테이징 영역과 작업 디렉토리 모두 HEAD 상태로 복구
- `git restore --source=HEAD^ --staged [file]`  
  - 스테이징 영역을 HEAD^ 상태로 복구
