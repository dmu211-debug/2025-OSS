# 오픈소스소프트웨어(OSS) 원격 수업 22~24 정리

## 22. 다양한 브랜치 병합(Merge)

### 병합(Merge)
- 두 개의 브랜치를 하나로 합치는 과정
  - **fast-forward(빠른 감기) 병합**
  - **3-way 병합**

### fast-forward 병합
- 조건
  - 현재 브랜치(master)가 병합할 대상 커밋의 직접적인 뿌리일 경우
  - 두 브랜치가 일렬 상태일 때 발생

### 3-way 병합
- 새로운 커밋을 생성하여 두 브랜치의 기록을 합침
- 브랜치가 갈라져 있는 상태에서 주로 사용

### 병합 명령과 옵션
- 일반 병합
  ```bash
git merge {병합할_브랜치_명}
- 브랜치가 일렬 상태 : fast-forward 병합
- 브랜치가 갈라져 있는 상태 : 3-way 병합
- git merge --no-ff {병합할_브랜치_명}  # 무조건 3-way 병합
- git merge --ff-only {병합할_브랜치_명} # fast-forward인 경우만 병합
- git merge --squash {병합할_브랜치_명} # 커밋 하나로 합치고 브랜치는 유지

## 24. 병합 충돌과 해결
### 병합 충돌 기준
- 한쪽 브랜치만 수정한 경우 : 충돌 없음
- 두 브랜치 모두 다른 변경 사항 발생 시 : 충돌 발생

### 3-way 충돌 예시
- git merge hotfix

### 충돌 해결 방법
1. 충돌 발생 파일 직접 수정
- 충돌 표시 제거 후 저장
2. 수정 후 스테이징 및 커밋
- git add <파일명>
- git commit -m '충돌 해결 메시지'
3. 필요 시 이전 브랜치 삭제 후 관리

## 병합 취소 후 다시 병합
- git merge --abort   # 병합 취소
- git merge feat/list # 다시 병합
